                         API ENDPOINTS (7 TOTAL)

PUBLIC (No Authentication):
   POST   /auth/register           Create account
   POST   /auth/login              Get JWT token

PROTECTED (Requires Authorization header with JWT):
   GET    /auth/me                 Get current user
   GET    /servers                 List all user servers
   POST   /servers                 Create new server
   PATCH  /servers/:id             Update server
   DELETE /servers/:id             Delete server

================================================================================
                        DATABASE SCHEMA (2 MODELS)
================================================================================

User Model:
   - id (Int, Primary Key, Auto-increment)
   - email (String, Unique)
   - password (String, Hashed with bcrypt)
   - name (String)
   - createdAt (DateTime)
   - servers (Relation to Server[])

Server Model:
   - id (Int, Primary Key, Auto-increment)
   - name (String)
   - host (String)
   - user (String, Default: "ubuntu")
   - port (Int, Default: 22)
   - env (String, Values: "prod"|"live"|"qa"|"test")
   - notes (String, Optional)
   - deploy (String, Optional)
   - logpath (String, Optional)
   - status (String, Default: "unknown")
   - uptime (Float, Default: 0)
   - createdAt (DateTime)
   - updatedAt (DateTime)
   - ownerId (Int, Foreign Key to User)
   - owner (Relation to User)

Relationships:
   - User has many Servers (one-to-many)
   - Servers belong to one User
   - Cascade delete: Deleting user deletes all their servers
